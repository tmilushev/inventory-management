Factory Inventory Management

System architecture for the full-stack demo application with Vue 3 frontend, FastAPI backend, and in-memory mock data.

Tech Stack

Frontend

  • Vue 3 UI framework
  • Composition API State & logic
  • Vue Router 4 SPA routing
  • Axios HTTP client
  • Vite 5 Build & dev server
  • Custom SVG Charts

Backend

  • FastAPI Web framework
  • Uvicorn ASGI server
  • Pydantic 2 Validation
  • Python 3.11+ Runtime
  • pytest + httpx Testing

Data Layer

  • JSON files (7) Storage
  • In-memory Runtime store
  • mock_data.py Loader
  • No database By design

System Architecture

Request Data Flow

1
User Action
Select filter in FilterBar or navigate to view
2
Composable
useFilters() refs update, watchers fire in views
3
API Call
api.js sends GET with query params via Axios
4
Filter
apply_filters() + filter_by_month() on in-memory data
5
Validate
Pydantic serializes response models
6
Render
Vue reactivity updates DOM with new data

Frontend Views

Dashboard

/

KPI cards, order health donut, orders by month bar chart, inventory value chart, shortages and top products tables.

Inventory

/inventory

Searchable stock levels table with SKU, quantity, reorder point, unit cost, and detail modals.

Orders

/orders

Status stat cards (4 statuses), orders table with expandable line items and delivery tracking.

Spending

/spending

Revenue/cost/profit cards, revenue vs costs chart, monthly cost flow, category breakdown pie, transactions.

Demand

/demand

Trend cards (increasing/stable/decreasing), demand forecasts table with current vs forecasted demand.

Reports

/reports

Quarterly performance table, monthly revenue trend chart, month-over-month growth analysis.

API Endpoints

MethodPathFiltersReturns
GET /api/inventory warehousecategory List of inventory items with stock levels
GET /api/inventory/{id} Single inventory item by ID
GET /api/orders warehousecategorystatusmonth List of orders with line items
GET /api/orders/{id} Single order by ID
GET /api/dashboard/summary warehousecategorystatusmonth Computed KPIs: inventory value, low stock, pending orders, totals
GET /api/demand Demand forecasts with trend indicators
GET /api/backlog Backlog items with computed purchase order flags
GET /api/spending/summary Procurement, operational, labor, overhead totals
GET /api/spending/monthly 12 months of spending by category
GET /api/spending/categories Spending breakdown by category with percentages
GET /api/spending/transactions Recent transaction history
GET /api/reports/quarterly Computed quarterly stats from orders
GET /api/reports/monthly-trends Month-over-month order and revenue analysis
POST /api/purchase-orders Creates purchase order for backlog item
